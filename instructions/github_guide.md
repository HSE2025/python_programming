# Руководство по работе с GitHub для групповых проектов

## Часть 1: Создание репозитория и добавление участников

### Шаг 1: Создание репозитория (делает ОДИН человек из группы)

1. Зайдите на [github.com](https://github.com) и войдите в свой аккаунт
2. Нажмите зелёную кнопку **New** (или **+** в правом верхнем углу → New repository)
3. Заполните:
   - **Repository name**: название проекта (например, `project-group-5`)
   - **Description**: краткое описание (необязательно)
   - **Public** или **Private**: выберите Private, если не хотите, чтобы код был виден всем
   - ✅ Поставьте галочку **Add a README file**
4. Нажмите **Create repository**

### Шаг 2: Добавление участников группы

1. В созданном репозитории нажмите **Settings** (вверху)
2. В левом меню выберите **Collaborators** (или Access → Collaborators)
3. Нажмите **Add people**
4. Введите GitHub username или email каждого участника группы
5. Нажмите **Add to this repository**
6. Участники получат приглашение на email — им нужно его принять!

## Часть 2: Установка Git

### Windows

**Вариант А: Git for Windows (рекомендуется)**
1. Скачайте с [git-scm.com/download/win](https://git-scm.com/download/win)
2. Запустите установщик
3. Нажимайте **Next** на всех шагах (настройки по умолчанию подходят)
4. После установки откройте **Git Bash** (появится в меню Пуск)

**Вариант Б: GitHub Desktop (проще, с интерфейсом)**
1. Скачайте с [desktop.github.com](https://desktop.github.com)
2. Установите и войдите в свой GitHub аккаунт

### macOS

**Вариант А: Через Terminal**
1. Откройте **Terminal** (Finder → Программы → Утилиты → Terminal)
2. Введите: `git --version`
3. Если Git не установлен, система предложит установить — нажмите **Install**

**Вариант Б: GitHub Desktop**
1. Скачайте с [desktop.github.com](https://desktop.github.com)
2. Установите и войдите в свой GitHub аккаунт

## Часть 3: Первоначальная настройка Git

Откройте терминал (Git Bash на Windows или Terminal на Mac) и выполните:

```bash
git config --global user.name "Ваше Имя"
git config --global user.email "ваш.email@example.com"
```

Используйте ваше GitHub account name как user.name и тот же email, что и в GitHub аккаунте!

## Часть 4: Клонирование репозитория (для ВСЕХ участников)

### Через терминал

1. На странице репозитория нажмите зелёную кнопку **Code**
2. Скопируйте HTTPS ссылку (например, `https://github.com/username/project-group-5.git`)
3. В терминале перейдите в папку, где хотите хранить проект:
   ```bash
   cd ~/Documents
   ```
4. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/username/project-group-5.git
   ```
5. Перейдите в папку проекта:
   ```bash
   cd project-group-5
   ```

### Через GitHub Desktop

1. File → Clone repository
2. Выберите репозиторий из списка или вставьте URL
3. Выберите папку для сохранения
4. Нажмите **Clone**

## Часть 5: Добавление существующего кода в репозиторий

⚠️ **ВАЖНО**: Если у вас уже есть код проекта, НЕ удаляйте его! Следуйте этим шагам:

### Шаг 1: Скопируйте ваши файлы в папку репозитория

1. Откройте папку, куда вы клонировали репозиторий
2. Скопируйте туда все файлы вашего проекта (кроме папки `.git`, если она есть)

### Шаг 2: Добавьте файлы в Git и отправьте на GitHub

```bash
# Посмотрите, какие файлы изменились
git status

# Добавьте все файлы
git add .

# Создайте коммит с описанием
git commit -m "Добавлен начальный код проекта"

# Отправьте на GitHub
git push
```

## Часть 6: Ежедневная работа с Git

### Основные команды (терминал)

```bash
# 1. ПЕРЕД началом работы — получите последние изменения от коллег
git pull

# 2. Работайте над кодом...

# 3. Посмотрите, что изменилось
git status

# 4. Добавьте изменения
git add .

# 5. Создайте коммит с понятным описанием
git commit -m "Добавлена функция загрузки данных"

# 6. Отправьте на GitHub
git push
```

### Через GitHub Desktop

1. **Перед работой**: нажмите **Fetch origin**, затем **Pull origin**
2. **После изменений**:
   - Слева увидите список изменённых файлов
   - Внизу напишите описание (Summary)
   - Нажмите **Commit to main**
   - Нажмите **Push origin**

## Часть 7: Решение типичных проблем

### Проблема: "Please tell me who you are"

Выполните команды настройки из Части 3.

### Проблема: Конфликт при pull (CONFLICT)

Это значит, что вы и коллега изменили один и тот же файл.

1. Откройте файл с конфликтом
2. Найдите строки вида:
   ```
   <<<<<<< HEAD
   ваш код
   =======
   код коллеги
   >>>>>>>
   ```
3. Оставьте нужный вариант, удалите маркеры `<<<<<<<`, `=======`, `>>>>>>>`
4. Сохраните файл
5. Выполните:
   ```bash
   git add .
   git commit -m "Resolved merge conflict"
   git push
   ```

### Проблема: Забыли сделать pull перед push

```bash
git pull --rebase
git push
```

### Проблема: Хочу отменить последние изменения (до коммита)

```bash
# Отменить изменения в одном файле
git checkout -- имя_файла.py

# Отменить все изменения
git checkout -- .
```

## Золотые правила работы в команде

1. **ВСЕГДА делайте `git pull` перед началом работы** — получите изменения коллег
2. **Делайте коммиты часто** — маленькие коммиты легче понять и откатить
3. **Пишите понятные сообщения коммитов** — "Исправлена ошибка в расчёте среднего" лучше, чем "fix"
4. **Не забывайте делать `git push`** — иначе коллеги не увидят ваши изменения
5. **Договоритесь, кто над чем работает** — это уменьшит количество конфликтов
