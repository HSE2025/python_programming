# Неделя 9 — Знай и люби свой компьютер 

---

## Давайте проверим Python прямо сейчас

**Windows:**
```cmd
python --version
```

**macOS:**
```bash
python3 --version
```
Скиньте свою версию в чатик!

---

## Три способа работы с Python

1. **REPL (интерактивный режим)** 
   → Быстрые эксперименты

2. **Скрипты (.py файлы)** 
   → Полноценные программы

3. **Google Colab** 
   → Онлайн (для сравнения)

**Каждый способ для своих задач!**

---

## Способ 1: Интерактивный режим (REPL)

**REPL = Read-Eval-Print-Loop**

**Запуск:**
```bash
# Windows
python

# macOS
python3
```

**Появится приглашение:**
```python
Python 3.14.0 (main, ...)
>>>
```

**Символы `>>>` = Python ждёт вашей команды!**

---

## Примеры в REPL

**Попробуйте набрать:**

```python
>>> print("Привет!")
Привет!

>>> 2 + 2
4

>>> text = "Божественная комедия"
>>> len(text)
21

>>> exit()  # выход
```

---

## Когда использовать REPL?

**Удобно для:**
- Быстрой проверки идей
- Тестирования одной команды
- Экспериментов с синтаксисом
- Проверки: как работает функция

**НЕ подходит для:**
- Длинных программ
- Сохранения кода
- Повторного использования

**REPL = Калькулятор для Python**

---

## Способ 2: Запуск скрипта (.py файл)

**Создадим файл `test_script.py`:**

```python
# test_script.py
print("Этот скрипт запущен из командной строки!")
```

**Запуск:**
```bash
# Windows
python test_script.py

# macOS
python3 test_script.py
```

---

## REPL vs Скрипт

| REPL (Интерактивный) | Скрипт (.py файл) |
|----------------------|-------------------|
| Каждая команда сразу | Весь код целиком |
| Результат мгновенный | Выполнение сверху вниз |
| Не сохраняется | Сохранён в файле |
| Для экспериментов | Для работы |
| Одна задача | Много операций |

**Оба способа полезны для разных целей!**

---

## Способ 3: Google Colab (облачные Jupyter notebooks)

| Локальный Python | Google Colab |
|------------------|--------------|
| Требует установки | Работает в браузере |
| Работает оффлайн | Нужен интернет |
| Без ограничений | Лимит 12 часов |
| Прямой доступ к файлам | Загрузка файлов |
| Серьёзные проекты | Обучение/эксперименты |

---

## Когда что использовать?

**REPL (интерактивный):**
- Проверить синтаксис, протестировать функцию, быстрый эксперимент

**Скрипты (.py):**
- Анализ корпуса текстов, обработка файлов, переиспользуемый код

**Google Colab:**
- Работа на чужом компьютере, быстрый эксперимент без установки, делиться кодом с коллегами

**Бесплатный** и open source

## Важные элементы кода

**1. Docstring (документация):**
```python
def count_words(text):
    """
    Подсчитывает количество слов в тексте
    """
```
- Описание в тройных кавычках `"""`
- VSCode покажет при наведении мыши

**2. `if __name__ == "__main__":`**
- Код выполняется только при прямом запуске
- Не выполняется при импорте как модуль

## Чтение файла в Python

**Создаём новый файл `file_reader.py`:**

```python
# file_reader.py
def read_file(filename):
    """Читает содержимое файла"""
    with open(filename, 'r', encoding='utf-8') as file:
      content = file.read()
      return content
```

**Важные элементы:**
- `with open(...)` — автоматически закрывает файл после выполнения блока кода
- `encoding='utf-8'` — правильная кодировка
---

## Важно: encoding='utf-8'

**БЕЗ правильной кодировки:**
```python
with open('file.txt', 'r') as file:  # ❌ Плохо
    content = file.read()
# Результат: ������������ �������
```

**С правильной кодировкой:**
```python
with open('file.txt', 'r', encoding='utf-8') as file:  # ✅ Хорошо
    content = file.read()
# Результат: Божественная комедия
```

**Всегда указывайте `encoding='utf-8'` для русского!**

## Функции анализа текста

**Добавим в `file_reader.py`:**

```python
def count_lines(text):
    """Подсчитывает непустые строки"""
    lines = text.split('\n')
    lines = [line for line in lines if line.strip()]
    return len(lines)


def count_words_in_text(text):
    """Подсчитывает слова"""
    words = text.split()
    return len(words)
```

## Проблема 1: VSCode не видит Python

## Проблема 2: FileNotFoundError

```
FileNotFoundError: [Errno 2] No such file or directory: 
'dante_text.txt'
```

**Причина:** Файл не в той папке что скрипт

**Решение:**
1. Убедиться что оба файла в одной папке
2. В VSCode: **File Explorer** слева покажет структуру
3. Перетащить файл в нужную папку
4. Или использовать полный путь:
   ```python
   filename = r"C:\Users\Maria\Desktop\dante_text.txt"
   ```

## Проблема 3: Кракозябры вместо текста

**Причина:** Неправильная кодировка

**Решение:**
```python
# ❌ ПЛОХО
with open('file.txt', 'r') as file:
    content = file.read()

# ✅ ХОРОШО
with open('file.txt', 'r', encoding='utf-8') as file:
    content = file.read()
```

**Всегда указывайте `encoding='utf-8'`!**
