## Язык программирования — язык естественный или язык формальный?
- **Алфавит**: символы (буквы/цифры/знаки), пробелы, переносы
- **Слова**: *токены* — имена (идентификаторы), ключевые слова, литералы
- **Предложения**: *операторы/выражения*; блоки формируются отступами
- **Пунктуация**: `() [] {} , : . ' " #` — скобки, запятая, двоеточие, точка, кавычки, комментарий
- **Грамматика**: правила соединения — синтаксис (отступы, двоеточия, скобки, порядок)

```python
# слово (имя)  # оператор   # литерал
year           =            1866
```

## Слайд 3. Алфавит и пунктуация Python (минимум на сегодня)
- Скобки: `()`, `[]`, `{}`; кавычки `' "` — строки; `#` — комментарий до конца строки
- Двоеточие `:` начинает блок (увидим позже, в `if`, `for`, `def`)
- Операторы: `+ - * / // % **`, `=`, `== != < > <= >=`, `and or not`, `in`, `is`
- Перевод строки завершает инструкцию; длинные выражения переносим в скобках
- Отступы **значимы**: не смешиваем табы с пробелами (используем 4 пробела)

## Переменная: имя и значение
- **Имена (идентификаторы)**: `snake_case`, латиница/цифры/`_`, не начинают с цифры
- **Значения (литералы)**: числа `42`, `3.14`, строки `"текст"`, булевы `True/False`, `None`

```python
title = "Гарри Поттер"
```

## Имена переменных: **правописание**
- `snake_case`: `book_title`, `total_words`, `is_valid`
- Понятные, короткие, без транслита: **лучше** `year`, чем `god`/`год`
- **Не** используем пробелы/дефисы/точки; **не** начинаем с цифры
- **Не затираем** встроенные имена: `list`, `dict`, `sum`, `len`, `str`, `type`

```python
good_name = 10
# sum = 5          # ❌ плохо: перекрывает встроенную функцию sum()
# str = "текст"    # ❌ потом str(10) перестанет работать
```

## Переменная и значение: модель «имя → объект»
**Идея:** имя — это ярлык (стрелка) на объект в памяти. Присваивание **не копирует** объект, а «приклеивает ярлык».
```html
Имена           Память (объекты)
-----           -----------------
year  ───────▶  1866 (int)

year = "1866"   # теперь имя указывает на другой объект
year  ───────▶  "1866" (str)

a = [1, 2]
b = a           # оба имени ссылаются на ОДИН список
a ─────┐
       └──────▶ [1, 2] (list)
b ─────┘
b.append(3)     # меняем объект — видят оба: a == [1,2,3], b == [1,2,3]
```
---
- **Изменяемые** объекты: `list`, `dict`, `set` — операции меняют сам объект
- **Неизменяемые**: `int`, `float`, `bool`, `str`, `tuple`, `NoneType` — операции создают **новый** объект, а имя перенаправляют
- Проверить «тот же объект?» → `id(a) == id(b)` (демо в Colab)

```python
a = [1,2]; b = a
id(a) == id(b)   # True
b.append(3); a   # [1,2,3]
x = 10; y = x; y += 1
x, y             # 10, 11 (y теперь указывает на НОВЫЙ int)
```

## Слайд 6. Присваивание и обновление
- `=` связывает имя с объектом; Python — динамически типизированный
- Множественное присваивание и обмен значениями
- Сокращённые операции: `+=`, `-=`, `*=`, `/=`, `//=`, `%=`

```python
a, b = 1, 2
a, b = b, a
score = 10
score += 5   # 15
```

## Слайд 7. Типы
- **int** — целые: `-3, 0, 42`
- **float** — вещественные: `3.14, -0.5`
- **bool** — логические: `True`, `False` (результат сравнений)
- **NoneType** — отсутствие значения: `None`
- **str** — строки (Unicode): `"текст"`
- **list** — список (изменяемый, упорядоченный): `[1, 2, 3]`
- **tuple** — кортеж (неизменяемый, упорядоченный): `(1, 2, 3)`
- **dict** — словарь (изменяемый, отображение): `{"author": "Пушкин"}`
- **set** — множество (изменяемое, без дубликатов): `{"кот", "код"}`
- **range** — арифм. прогрессия: `range(5)` → `0..4`

## Слайд 9. Выражения и операции
- Арифметика: `+ - * /` (деление всегда float), `//` (целочисл.), `%` (остаток), `**` (степень)
- Строки: конкатенация `"a" + "b"`, умножение `"la"*3`
- Сравнения: `== != < > <= >=` → `bool`
- Приоритет как в математике; скобки — ваш друг

```python
2 + 2 * 2        # 6
"год: " + str(1866)
```

## Базовые встроенные функции (минимум)
- `print(x, ...)` — вывод; параметры: `sep`, `end`
- `len(x)` — длина (строка/список/словарь/множество…)
- `type(x)` — тип
- Приведение типов: `int("42")`, `float("3.14")`, `str(10)`, `bool("")`
- Числовые: `abs(x)`, `round(x[, n])`, `min`, `max`, `sum`

```python
print("a","b","c", sep="|", end="!")
len("Привет"), type(3.0), int("42")
```

## Анти-капкан: частые ошибки
1) **Перекрытие встроенных имён**: `list = []`, `sum = 0`, `str = "..."` → ломает одноимённые функции  
2) **Смешивание типов**: `"1866" + 10` → `TypeError` (нужно `int("1866")`)  
3) **Неправильные кавычки/скобки**: забытая кавычка → `SyntaxError`  
4) **Отступы**: смешали табы/пробелы → `IndentationError` (ставим 4 пробела)  
5) **Деление**: `/` даёт `float` (даже `4/2 == 2.0`), для целого — `//`  
6) **`int("42.0")`** → `ValueError` (сначала `float("42.0")`, потом `int()`)  
7) **Unicode-нюансы**: длина в символах ≠ длина в байтах
---
```python
# Плохо:
# sum = 5
# print(sum([1,2,3]))  # теперь упадёт: sum не функция

# Хорошо:
total = 5
print(sum([1,2,3]))
```

## Чтение ошибок (traceback) — быстрый алгоритм
- Смотрим **последнюю строку**: тип/сообщение (`TypeError: ...`)
- На строку выше смотрим ↑ — там место ошибки
- Ищем ключевые слова из сообщения (гугл/LLM)
- Повторяем запуск после правки

```python
year = "1866"
# print(10 + year)           # TypeError
print(10 + int(year))        # ✔ фикс
```

## Чек-лист синтаксиса Python
- [ ] 4 пробела на отступ; не смешивать табы и пробелы
- [ ] После `if/for/while/def/class` ставим `:` и **отступаем** следующую строку
- [ ] Строки в кавычках `'...'` или `"...";` многострочные — `'''...'''`/`"""..."""`
- [ ] Комментарии начинаются с `#` (до конца строки)
- [ ] **Имена:** `snake_case` для переменных/функций, `PascalCase` для классов, `UPPER_SNAKE_CASE` для констант; **избегаем** `camelCase` в своём коде
- [ ] `=` — присваивание, `==` — сравнение (не путать!)
